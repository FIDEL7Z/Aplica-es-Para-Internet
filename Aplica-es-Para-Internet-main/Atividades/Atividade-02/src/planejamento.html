📐 Planejamento do Projeto

1. Definição do Problema

Contexto: Desenvolvedores precisam apresentar suas habilidades de forma profissional.

Público-alvo: Recrutadores e professores.

Dor: Dificuldade de visualizar informações rapidamente.

Sucesso: Entender o perfil em menos de 10 segundos.

2. Wireframe

⬇️ Baixar imagem do Wireframe

3. Arquitetura de Informação

Organização: Conteúdo dividido em seções claras: Sobre, Experiência, Projetos e Contato.
Cada seção com propósito específico

Rotulagem: Nomes intuitivos: Sobre, Experiência, Projetos, Contato

Navegação: Navegação por menu com âncoras e Rolagem direta para seções

Hierarquia: H1 → Nome principal, Títulos de seção → destaque, Texto → suporte

4. Tipografia

Famílias utilizadas: Títulos: Syne, Texto: DM Sans

Hierarquia: H1: Nome principal, Títulos: Seções, Texto: descrições

Objetivo: Garantir legibilidade e Criar hierarquia visual clara

5. Sistema de Cores

Cores principais: Fundo: escuro (#0d0f14), Destaque Jefferson: azul, Destaque Hugo: roxo

Funções: Diferenciar perfis e Guiar atenção do usuário

Acessibilidade: Contraste adequado para leitura

6. Organização CSS

Componentes reutilizáveis: Remoção de margens e paddings, box-sizing: border-box

Organização: Uso de variáveis por CSS e Separação por componentes

Estrutura lógica: Hero, Navegação, Seções, Cards

Componentes reutilizáveis:
→ profile-card (perfil dos integrantes)
→ project-card (projetos)
→ contact-card (contato)

7. Responsividade

Layout adaptável usando Grid e Flexbox.

No mobile, colunas viram uma única coluna.

Técnicas usadas:
→ CSS Grid (organizar layout)
→ Flexbox (alinhar elementos)
→ Media Queries (adaptar telas)

Comportamento:
→ Desktop: 2 colunas lado a lado
→ Mobile: tudo empilhado
→ Texto se adapta com clamp()
